## Data quality findings (roadblocks worked through today)

1. **Duplicate ARM column.** The main CSV had two columns both named
   "ARM" on export; R auto-renamed the second to `ARM.1`. Confirmed via
   `identical()` that the values were exactly duplicated, then dropped
   `ARM.1`. No information lost.

2. **PATNO mismatch between files.** Initial patient-count check showed
   `mh` contains 1,433 unique patient IDs — 132 more than the 1,301 in
   the trial's main dataset. Investigated whether this was a data
   formatting bug (e.g. leading zeros being dropped) — ruled that out,
   since all mismatched numeric IDs are the same uniform 6-character
   length as everything else. Conclusion: 142 PATNO values in `mh` don't
   belong to any of the 1,301 trial patients:
   - 124 are well-formed 6-digit IDs — likely medical history collected
     at trial *screening*, before randomization, meaning these may be
     patients who were screened but never actually enrolled in the
     1,301-patient analysis population.
   - 18 have an unusual 18-character format, clearly different from the
     standard ID pattern — cause unknown, only 18 rows out of 8,991 so
     low impact regardless of explanation.

   No fix was needed for downstream work — the planned join (comorbidity
   matrix onto main dataset) uses `main` as the base table, so these
   142 non-trial IDs will be automatically and correctly excluded.

3. **MHONGO/MHTERM overlap check.** Of the 38 missing MHONGO values, 21
   are on fully blank rows (harmless), but 17 are on rows with a real
   MHTERM entry and no ongoing/not-ongoing flag recorded. Working
   assumption: treat missing MHONGO the same as MHONGO = 1 (count toward
   comorbidity burden) since it can't be ruled out. Flagged for mentor
   review since this is an assumption, not a confirmed rule.
